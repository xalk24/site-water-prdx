# Деплой лендинга «Вода в такси»

Проект — полностью статический сайт без сборщика, backend, CDN и внешних runtime-зависимостей. Корень репозитория является готовым выходным каталогом.

## Состав

- `index.html` — страница;
- `assets/voda-v-taxi/styles.css` — стили;
- `assets/voda-v-taxi/app.js` — progressive enhancement для появления секций;
- `assets/voda-v-taxi/*.svg` — локальные favicon и Open Graph обложка;
- `robots.txt` — правила индексации;
- `brand-spec.md` — статус брендовых ассетов и ограничений.

## Локальная проверка

Из корня проекта:

```bash
python3 -m http.server 8080
```

Откройте `http://localhost:8080`. Открытие `index.html` напрямую также работает, потому что все пути относительные.

## Обычный static hosting

Загрузите содержимое корня проекта в document root хостинга (`public_html`, `www` или аналог). Для Netlify, Cloudflare Pages, GitHub Pages и S3 укажите корень репозитория как publish directory; build command не нужен. Для хостинга, требующего каталог `dist`, скопируйте в него `index.html`, `robots.txt` и каталог `assets` без изменения структуры.

После привязки домена:

1. Включите HTTPS и редирект с HTTP.
2. Добавьте в `<head>` абсолютный `canonical` с production-доменом.
3. Замените `og:image` на абсолютный URL.
4. Создайте `sitemap.xml` с production-URL и добавьте его адрес в `robots.txt`.
5. Проверьте страницу на 320, 375, 768, 1024 и 1440 px.

## Форма и контакты

Форма намеренно отключена: она не имитирует отправку и не передаёт данные. Сейчас посетителю доступны `tel:` и `mailto:`. Чтобы включить отправку:

1. Подготовьте политику конфиденциальности и текст согласия.
2. Добавьте обязательный checkbox со ссылкой на политику.
3. Подключите собственный HTTPS endpoint или форму static-хостинга.
4. Реализуйте состояния loading, success и error; не помещайте секреты/API-ключи в браузерный код.
5. Проверьте серверную валидацию, защиту от спама и место хранения данных.

## Аналитика

Аналитика и cookies не подключены. Подключайте их только после определения consent-модели и политики конфиденциальности. Любую внешнюю сеть зафиксируйте здесь.

## Замена ассетов

Официальный логотип и реальные фотографии не были предоставлены. После их получения храните оптимизированные варианты локально в `assets/voda-v-taxi/`, используйте AVIF/WebP с JPEG fallback, задайте `width`/`height`, `srcset` и корректный `alt`. Не публикуйте изображения из исходного PDF без подтверждения прав.

## Rollback

Храните предыдущий релиз как отдельную версию или git tag. Для отката повторно опубликуйте содержимое предыдущего коммита; база данных и миграции отсутствуют.
